# 贡献指南 | Contributing Guide

感谢你对「文明的原理」项目的兴趣。

---

## 参与方式

### 1. 贡献新的"XX的原理"

选择一个你熟悉的领域，创建一个新目录，按照统一方法开始写作。

**步骤：**

1. Fork 本仓库
2. 创建目录：`你的领域的原理/`（如 `医学的原理/`、`心理学的原理/`）
3. 在目录下创建 `README.md`，写明：
   - **母句**：这个领域到底在解释什么（一句话）
   - **核心问题**：普通人在这个领域最常感到困惑的 5-10 个问题
   - **卷/章规划**：初步的结构安排
4. 按 `_templates/` 中的模板开始写作
5. 提交 Pull Request

### 2. 改进现有内容

- 修正事实错误
- 改善表达的可感性和可进入性
- 补充经验锚点（具体例子、日常类比）
- 加强跨卷/跨领域的呼应

### 3. 翻译

- 在对应目录下创建 `translations/` 子目录
- 文件名加语言后缀，如 `第一卷.en.md`

### 4. 讨论与反馈

- 在 Issues 中提出问题、质疑、建议
- 使用标签：`内容讨论`、`方法论`、`新领域提案`、`翻译`

## 写作原则

所有内容必须遵循项目的核心方法论（详见 `_methodology/从高密度到中高密度.md`）：

### 必须做到

- [ ] **从人的经验入口进入**，不从术语定义起步
- [ ] **保留原理骨架**，不为通俗牺牲结构
- [ ] **每一章有具体的经验锚点**（至少一个可感的例子）
- [ ] **每一章的核心论证可回译到严肃理论**
- [ ] **语言清晰、节制、有力量**，不堆辞藻

### 必须避免

- ❌ 学术黑话堆砌（不解释就使用专业术语）
- ❌ 低密度泛科普（只讲故事不见结构）
- ❌ 鸡汤化（用情绪替代论证）
- ❌ 百科全书式罗列（缺少主线贯穿）

---

## 文件规范

- 所有正文使用 **Markdown** 格式
- 编码：**UTF-8 with BOM**
- 一个"XX的原理"对应一个顶层目录
- 每卷一个 `.md` 文件，文件名即卷标题
- 图片放在对应目录的 `assets/` 子目录中

---

## Pull Request 检查清单

提交 PR 时，请确认：

- [ ] 内容遵循转译方法论（体验→困惑→结构→原理→回到生活）
- [ ] 不含抄袭或版权侵犯内容
- [ ] 文件放在正确目录下
- [ ] Markdown 格式正确，无渲染错误
- [ ] 如果是新领域，已创建 README.md 说明母句和规划

---

## 行为准则

- 尊重不同知识背景的贡献者
- 批评对事不对人，聚焦内容质量
- 鼓励跨学科对话
- 以"让更多人理解世界"为共同目标
